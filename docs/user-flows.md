# User Flows

## Public Browse and Apply
1. Visitor lands on the home page.
2. Visitor searches jobs or browses open vacancies.
3. Visitor opens a vacancy detail page to read more.
4. Visitor clicks apply.
5. If not authenticated, the visitor is taken to login or registration.

## Locum Onboarding
1. Register as locum.
2. Verify email (foundation event).
3. Create the first resume/CV with required personal, professional, and document details.
4. Optionally create additional resumes for different specialties or presentation styles.
5. Submit required information for verification.
6. Once approved, apply to active jobs by selecting a saved resume and adding a short message.

## Clinic Onboarding
1. Register as clinic.
2. Complete facility profile and upload verification docs.
3. Submit verification.
4. Admin approves/rejects.
5. Approved clinic posts opportunities.

## Application Workflow
1. Locum opens a vacancy and clicks apply.
2. Locum selects which saved resume to use.
3. Locum writes a short message.
4. Application is submitted with a resume snapshot.
5. Clinic receives an in-app notification and email notification.
6. Clinic reviews the application, opens the resume, messages the applicant, and moves the application through stages.
7. Locum receives status notifications.
8. History of status transitions is recorded.

## Job Moderation
1. Clinic creates draft.
2. Clinic submits for review (or auto-publish policy).
3. Admin approves/rejects with note.
4. Active job appears in discovery.

## Suspension/Compliance
- Admin can suspend users, resumes, profiles, jobs, and documents.
- Suspended actors lose protected workflow access.
- Actions are audit logged.
